# @latimer-woods-tech/compliance

TCPA suppression list management, FDCPA 24-hour contact window enforcement, immutable consent records, and end-to-end Data Subject Request (DSR) lifecycle tracking. Depends on `@latimer-woods-tech/neon`.

## Installation

```bash
npm install @latimer-woods-tech/compliance
```

## Database

```typescript
import {
  CREATE_COMPLIANCE_CONSENTS_TABLE,
  CREATE_COMPLIANCE_CONTACTS_TABLE,
  CREATE_TCPA_SUPPRESSION_TABLE,
  CREATE_DSR_REQUESTS_TABLE,
  CREATE_DSR_ARTIFACTS_TABLE,
} from '@latimer-woods-tech/compliance';

await db.execute(CREATE_COMPLIANCE_CONSENTS_TABLE);
await db.execute(CREATE_COMPLIANCE_CONTACTS_TABLE);
await db.execute(CREATE_TCPA_SUPPRESSION_TABLE);
await db.execute(CREATE_DSR_REQUESTS_TABLE);
await db.execute(CREATE_DSR_ARTIFACTS_TABLE);
```

## API

### `checkTCPA(opts)`

Checks whether a phone number is on the TCPA suppression list.

```typescript
function checkTCPA(opts: {
  db: Db;
  phone: string;
}): Promise<{ safe: boolean; reason?: string }>;
```

### `suppressPhone(db, phone, reason?)`

Adds a phone number to the TCPA suppression list. Idempotent (`ON CONFLICT DO NOTHING`).

```typescript
function suppressPhone(db: Db, phone: string, reason?: string): Promise<void>;
```

### `logConsent(db, opts)`

Appends an immutable consent record. Records are never updated or deleted.

```typescript
function logConsent(db: Db, opts: {
  userId: string;
  consentType: 'TCPA' | 'FDCPA' | 'GDPR' | 'CCPA';
  ipAddress: string;
  userAgent?: string;
}): Promise<void>;
```

### `checkFDCPA(db, opts)`

Returns whether a contact attempt is permitted under FDCPA's 24-hour rule.

```typescript
function checkFDCPA(db: Db, opts: {
  contactId: string;
  callType: 'initial' | 'follow_up';
}): Promise<{ allowed: boolean; nextAllowedAt?: Date; reason?: string }>;
```

### `recordContact(db, opts)`

Inserts a contact record used by `checkFDCPA`.

```typescript
function recordContact(db: Db, opts: {
  contactId: string;
  callType: 'initial' | 'follow_up';
}): Promise<void>;
```

## DSR (Data Subject Request) API

These functions implement the G-15 compliance gap: end-to-end DSR lifecycle for the HumanDesign Practitioner tier.

### `submitDSR(db, opts)`

Submits a new Data Subject Request and returns its generated ID.

```typescript
function submitDSR(db: Db, opts: {
  userId: string;
  appId?: string;
  requestType: 'access' | 'erasure' | 'portability' | 'rectification';
  notes?: string;
}): Promise<string>;
```

### `getDSRStatus(db, requestId)`

Returns the current status and metadata of a DSR.

```typescript
function getDSRStatus(db: Db, requestId: string): Promise<DsrRequest>;
```

### `listDSRRequests(db, opts?)`

Lists DSR requests for the operator admin surface, optionally filtered.

```typescript
function listDSRRequests(db: Db, opts?: {
  userId?: string;
  appId?: string;
  status?: 'pending' | 'in_progress' | 'fulfilled' | 'rejected';
  limit?: number;
}): Promise<DsrRequest[]>;
```

### `fulfillDSR(db, opts)`

Records a fulfillment artifact and marks the DSR as `fulfilled`.

```typescript
function fulfillDSR(db: Db, opts: {
  requestId: string;
  kind: string;
  payload: Record<string, unknown>;
}): Promise<void>;
```

### `eraseDSR(db, userId)`

Enforces erasure semantics: anonymises PII in consent records and fulfils any open `erasure` DSRs for the user. Does not hard-delete rows.

```typescript
function eraseDSR(db: Db, userId: string): Promise<void>;
```

## Example

```typescript
import {
  checkTCPA,
  logConsent,
  checkFDCPA,
  recordContact,
  submitDSR,
  getDSRStatus,
  fulfillDSR,
  eraseDSR,
} from '@latimer-woods-tech/compliance';

// Before placing a call
const tcpa = await checkTCPA({ db, phone: '+15551234567' });
if (!tcpa.safe) {
  return c.json({ error: 'Phone suppressed' }, 403);
}

const fdcpa = await checkFDCPA(db, { contactId: contact.id, callType: 'initial' });
if (!fdcpa.allowed) {
  return c.json({ error: 'FDCPA window not open', nextAllowedAt: fdcpa.nextAllowedAt }, 429);
}

await recordContact(db, { contactId: contact.id, callType: 'initial' });

// DSR intake
const dsrId = await submitDSR(db, {
  userId: 'user-123',
  appId: 'humandesign',
  requestType: 'erasure',
});

// DSR fulfilment
await fulfillDSR(db, {
  requestId: dsrId,
  kind: 'erasure_manifest',
  payload: { erasedAt: new Date().toISOString(), tables: ['compliance_consents'] },
});

// Or enforce immediate erasure
await eraseDSR(db, 'user-123');
```
