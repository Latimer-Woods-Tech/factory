# 2026-05-02: CSP stale hash blocked FOUC script

**Severity:** P2  **Duration:** ~2 days  **Impact:** All selfprime.net users (theme/density not applying)

## What happened
PR #57 added density to the FOUC inline script. The SHA-256 hash in `client/public/_headers` was not updated. Every page load produced a CSP violation. Visual bugs: thin sidebar, floating bodygraph.

## Root cause
No coupling between "change inline script" and "update CSP hash."

## Timeline
- 2026-05-01: PR #57 merged
- 2026-05-02 20:30 UTC: First real user reports visual bugs
- 2026-05-02 21:15 UTC: PR #74 merged — 1-line fix

## What we changed
- Updated CSP hash (PR #74)
- Added CSP procedure to `security.md` playbook

CSP violations in browser console.
